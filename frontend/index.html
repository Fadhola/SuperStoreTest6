About the Dashboard Visualization

The Superstore Dashboard is designed to provide users with an interactive and comprehensive data visualization experience. Built using modern web technologies such as Vite, Express.js, and MongoDB, this dashboard allows businesses to analyze key sales and profit metrics efficiently.

About the Dataset

The Superstore dataset contains detailed records of sales and profits across products, regions, categories and customer segments. Datasets This is a valuable resource for aspiring businesses optimize their strategies and increase profitability.

Features

  • Dynamic data visualization using Chart.js for insightful graphical representation
  • Interactive data tables with DataTables for seamless filtering and sorting
  • Geospatial data visualization powered by Leaflet for location-based insights
  • Full CRUD functionality allowing users to add, edit, and manage datasets securely
  • Multi-user data management ensuring each user’s dataset remains private and protected
  • Advanced filtering options for tailored data analysis based on region, product category, and time frame
  • Responsive design for optimal experience across desktop, tablet, and mobile devices

Purpose

The goal of this dashboard is to empower businesses with real-time insights, enabling better decision-making through data-driven strategies. With customizable filters, user-specific data management, and intuitive visual analytics, businesses can optimize their operations and enhance profitability.

Metadata & Data Dictionary

Please ensure your CSV or JSON file follows the structure outlined below to successfully upload and validate your dataset. Fields are case-sensitive and must match exactly as shown. Each row in your dataset must contain data for all required fields.

          Row ID,Order ID,Order Date,Ship Date,Ship Mode,Customer ID,Customer Name,Segment,Country,City,State,Postal Code,Region,Product ID,Category,Sub-Category,Product Name,Sales,Quantity,Discount,Profit,Profit/Quantity

Fields and Description

Field Description Data Type
Row ID Unique ID for each row of data in the table. This field is used only for internal purposes in the CSV file to uniquely identify each row. However, it is not required and will be automatically removed during the upload process as it is not needed in the database schema. Integer (optional, will be removed)
Order ID Unique identifier for each order placed by the customer. String (non-empty)
Order Date Date the order was placed. Date (valid format: D/M/YYYY, MM/DD/YYYY, etc.)
Ship Date Date the product was shipped (must be later than the order date). Date (valid format)
Ship Mode The mode of shipping selected (e.g., Standard Class, Expedited). String (non-empty)
Customer ID Unique identifier for the customer. String (non-empty)
Customer Name Name of the customer. String (non-empty)
Segment Customer segment (must be one of: Consumer, Corporate, Home Office). String (must be one of: Consumer, Corporate, Home Office)
Country The country of residence of the customer. String (non-empty)
City The city of residence of the customer. String (non-empty)
State The state of residence of the customer. String (non-empty)
Postal Code Postal code of the customer. Number (valid)
Region The region where the customer belongs (e.g., East, West, Central, South). String (must be one of: East, West, Central, South)
Product ID Unique identifier for the product. String (non-empty)
Product Name Name of the product. String (non-empty)
Category Category of the product (e.g., Furniture, Office Supplies). String (non-empty)
Sub-Category Sub-category of the product (e.g., Chairs, Paper). String (non-empty)
Sales The total sales amount for the product. Number (valid)
Quantity The number of items ordered. Integer (greater than 0)
Discount The discount on the product (should be between 0 and 1). Number (0 to 1)
Profit The profit earned from the sale of the product. Number (valid)
Profit/Quantity The profit per quantity sold. Number (valid)

File Validation

During the upload process, your CSV or JSON file will be validated against the following criteria:

  • Field Presence: Each row must include all required fields (e.g., Order ID, Order Date, Ship Date, etc.). If any required field is missing, an error will occur.
  • Data Types: Fields must match the expected data types (e.g., Sales must be a number, Order Date must be a valid date).
  • Date Format: Dates must be in one of the acceptable formats: D/M/YYYY, MM/DD/YYYY, or M/D/YYYY. Invalid dates will trigger an error.
  • Field Length and Values: Ensure fields like Discount (should be between 0 and 1), and Category/Sub-Category contain valid options as listed.
