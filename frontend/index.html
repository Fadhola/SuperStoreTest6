About the Dataset

Dataset Superstore berisi catatan rinci tentang penjualan dan profit dari berbagai produk, wilayah, kategori, dan segmen pelanggan. Dataset ini merupakan sumber daya berharga bagi bisnis yang ingin mengoptimalkan strategi mereka dan meningkatkan profitabilitas.

Context

Dengan meningkatnya permintaan dan persaingan yang ketat di pasar, sebuah Superstore Giant mencari wawasan untuk memahami produk, wilayah, kategori, dan segmen pelanggan mana yang harus ditargetkan atau dihindari. Selain itu, terdapat peluang untuk membangun model prediksi untuk penjualan dan profit.

Metadata & Data Dictionary

Please ensure your CSV or JSON file follows the structure outlined below to successfully upload and validate your dataset. Fields are case-sensitive and must match exactly as shown. Each row in your dataset must contain data for all required fields.

Correct Format for CSV:

The dataset must have the following columns in the order listed below. Make sure there are no missing fields in any row. Here is an example of the first row in your CSV: 

          Row ID,Order ID,Order Date,Ship Date,Ship Mode,Customer ID,Customer Name,Segment,Country,City,State,Postal Code,Region,Product ID,Category,Sub-Category,Product Name,Sales,Quantity,Discount,Profit,Profit/Quantity

Make sure your data matches the following structure, and each row contains values for all required fields. If any required fields are missing, the file will not pass validation.

Fields and Description

Field Description
Row ID Unique ID for each row of data in the table. This field is used only for internal purposes in the CSV file to uniquely identify each row. However, it is not required and will be automatically removed during the upload process as it is not needed in the database schema.
Order ID Unique identifier for each order placed by the customer. Must be a non-empty string.
Order Date Date the order was placed (format: D/M/YYYY, MM/DD/YYYY, etc.). Must be a valid date.
Ship Date Date the product was shipped (must be in valid date format and should be later than the order date).
Ship Mode The mode of shipping selected (e.g., Standard Class, Expedited).
Customer ID Unique identifier for the customer (non-empty string).
Customer Name Name of the customer (non-empty string).
Segment Customer segment (must be one of: Consumer, Corporate, Home Office).
Country The country of residence of the customer.
City The city of residence of the customer.
State The state of residence of the customer.
Postal Code Postal code of the customer (must be a valid number).
Region The region where the customer belongs (e.g., East, West, Central, South).
Product ID Unique identifier for the product.
Product Name Name of the product (e.g., Global Deluxe Stacking Chair).
Category Category of the product (e.g., Furniture, Office Supplies).
Sub-Category Sub-category of the product (e.g., Chairs, Paper).
Sales The total sales amount for the product (must be a valid number).
Quantity The number of items ordered (must be an integer and greater than 0).
Discount The discount on the product (should be between 0 and 1).
Profit The profit earned from the sale of the product.
Profit/Quantity The profit per quantity sold (must be a valid number).

File Validation

During the upload process, your CSV or JSON file will be validated against the following criteria:

  • Field Presence: Each row must include all required fields (e.g., Order ID, Order Date, Ship Date, etc.). If any required field is missing, an error will occur.
  • Data Types: Fields must match the expected data types (e.g., Sales must be a number, Order Date must be a valid date).
  • Date Format: Dates must be in one of the acceptable formats: D/M/YYYY, MM/DD/YYYY, or M/D/YYYY. Invalid dates will trigger an error.
  • Field Length and Values: Ensure fields like Discount (should be between 0 and 1), and Category/Sub-Category contain valid options as listed.
